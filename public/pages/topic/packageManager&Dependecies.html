Package Manager & Dependencies

Maven vs. npm. How to define and install dependencies.

1. Maven

Maven is an open-source project that was initially created by Jason van Zyl in 2002. It is built using the Java programming language and is widely used by developers to manage and build their Java-based projects.One of the key features of Maven is its use of a Project Object Model (POM). The POM is an XML file that describes the project's configuration and dependencies. It includes information such as the project's name, version, and dependencies. The POM also specifies the build process for the project, including the build phases and goals.

One of the main benefits of using Maven is its dependency management system. Maven automatically downloads and manages dependencies for a project, ensuring that the correct versions are used and that conflicts are avoided. This makes it easier for developers to manage their projects and reduces the risk of errors caused by incompatible dependencies.

2. npm

npm stands for Node Package Manager. It is a package manager for Node.js packages, modules, and libraries. It is free and open-source, and it is used to manage the installation, updating, and removal of Node.js packages.

npm was created in 2009 as a part of the Node.js project. One of the key features of npm is its ability to manage dependencies between packages. npm automatically installs any package dependencies that are required by a package, ensuring that the correct versions are used and that conflicts are avoided. It also provides a mechanism for developers to specify the required version of a package, making it easy to ensure compatibility with existing code.

To install npm, you will need to first install Node.js.

After, you can open the command line interface and type

        npm install

As Maven has a specific file where all the dependencies are declared, npm has the 'package.json' file. It is a JSON file that contains information about the project, such as its name, version, author, license, and dependencies. The package.json file is located in the root directory of the project and is created automatically when you run npm init command in the terminal. The package.json file can also be created manually by adding the required fields as per the project's needs.

4. How to install packages

There are two options to install packages. In this case we will install Express

The first way to install express is to run the following command in the commandaline

        npm install express

After running the command, we can see files and folders in the root folder that confirm the correct installation of Express. npm generates a folder called 'node_modules' with inside the packages required by Express and two other json files: 'package.json' and 'package-lock.json'.

'package.json' is a file that is used in Node.js projects to manage dependencies. It is essentially a manifest file that lists all the dependencies that the project requires to run. These dependencies can include other libraries or packages that are needed by the project. In this file, you can specify the version of the dependency that you want to use as well as other important information like the author, license, and description of the project.

'package-lock.json', on the other hand, is a file that was introduced in npm version 5 and it ensures that the exact version of each dependency is installed. It also locks the version of each dependency so that it does not change unexpectedly. This is important because it ensures that the project will work consistently across different machines and environments.

The second way to install Express is to declare manually the dependency in the 'package.json' file.

First, create the 'package.json' file in the root folder of the project and the declare the dependcy 

        {
        "dependencies" : {
            "express": "4.18.2"
        }
    }

After we can simply run npm install and see the same results and consequences achieved beofore.
