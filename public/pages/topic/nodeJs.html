1. NodeJs

Node.js is a JavaScript runtime environment for developing server-side applications. It uses Google's V8 JavaScript engine, which compiles and executes JavaScript code. Node.js allows developers to write network applications using JavaScript, which is easier than using other languages like C++ or Java.

Node.js was created by Ryan Dahl in 2009 when he was working on a web application that required real-time communication between the browser and the server. He needed to create an efficient method of communicating between these two parts of the application, so he created Node.js and released it as open source software under the MIT license on GitHub. The popularity of Node.js grew quickly because of its low resource requirements, its ease of use and its ability to handle multiple concurrent connections efficiently (especially compared to other languages).

To run a file with Node.js, you need to use the 'node' command line tool.
The node command line tool is part of the Node.js installation package, which you can download from the official website of Node.js.

To run files with Node.js, simply open a terminal window on your computer or IDE and type in: 

    node [filename].js
2. Package Manager and Dependecies

npm stands for Node Package Manager. It is a package manager for Node.js packages, modules, and libraries. It is free and open-source, and it is used to manage the installation, updating, and removal of Node.js packages.

npm was created in 2009 as a part of the Node.js project. One of the key features of npm is its ability to manage dependencies between packages. npm automatically installs any package dependencies that are required by a package, ensuring that the correct versions are used and that conflicts are avoided. It also provides a mechanism for developers to specify the required version of a package, making it easy to ensure compatibility with existing code.

To install npm, you will need to first install Node.js.

After, you can open the command line interface and type

    npm install

npm has the 'package.json' file that is a JSON file that contains information about the project, such as its name, version, author, license, and dependencies. The package.json file is located in the root directory of the project and is created automatically when you run npm init command in the terminal. The package.json file can also be created manually by adding the required fields as per the project's needs.

3. How to install packages

There are two options to install packages. In this case we will install Express

The first way to install express is to run the following command in the commandaline

    npm install express

After running the command, we can see files and folders in the root folder that confirm the correct installation of Express. npm generates a folder called 'node_modules' with inside the packages required by Express and two other json files: 'package.json' and 'package-lock.json'.

'package.json' is a file that is used in Node.js projects to manage dependencies. It is essentially a manifest file that lists all the dependencies that the project requires to run. These dependencies can include other libraries or packages that are needed by the project. In this file, you can specify the version of the dependency that you want to use as well as other important information like the author, license, and description of the project.

'package-lock.json', on the other hand, is a file that was introduced in npm version 5 and it ensures that the exact version of each dependency is installed. It also locks the version of each dependency so that it does not change unexpectedly. This is important because it ensures that the project will work consistently across different machines and environments.

The second way to install Express is to declare manually the dependency in the 'package.json' file.

First, create the 'package.json' file in the root folder of the project and the declare the dependcy 

    {
    "dependencies" : {
        "express": "4.18.2"
    }
  }

After we can simply run npm install and see the same results and consequences achieved beofore.

4. Express

Express is a popular and widely used web framework for Node.js. It provides a set of tools and features for building robust and scalable web applications. Express is lightweight, flexible, and easy to use, making it a popular choice for developers who need to build web applications quickly.

Express is built on top of the core Node.js HTTP module, which means that it provides a higher-level API for handling HTTP requests and responses. It also provides a set of middleware functions that can be used to handle common tasks such as parsing request bodies, handling cookies, and handling sessions.

One of the key features of Express is its routing system. Express provides a powerful routing system that allows developers to define routes for different HTTP methods (GET, POST, PUT, DELETE, etc.) and URL patterns. This makes it easy to create RESTful APIs and other web services.

5. How to set a server

The first step is to install the package create the 'app.js' file in the root folder of the project. It is that it is the main file of the application. It contains the logic to initialize the application, including loading modules, setting up middleware, and defining routes. The app.js file is typically the first file that is executed when the application is started, and it is responsible for starting the server and listening for incoming requests. It is also responsible for handling errors and providing a response to the client. In short, the app.js file is the heart of the application, and it is where most of the application logic is defined.

After we can start using the package we previously isntalled by writing 

    const express = require("express");
    const app = express();
    app.use(express.json());

    //at the end of the file
    const PORT = 8080;
    app.listen(PORT, (error) => {
        error ? console.log(error) : console.log(
        "Server is running on port: ", PORT
        );
    });

We use the 8080 port because by convention it is the http developer port. Port 8000 is for https.

6. Anatomy of URL

https://localhost/kickdinosaur/:id?size=1200

• 'https://' is the protocol

•“localhost” is the domain

• “/kickdinosour” is the endpoint

• “:id” is the path variable, parameter (it is mandatory to reach the endpoint)

• “?size=1200” is the query string, query parameter (it is optional)
